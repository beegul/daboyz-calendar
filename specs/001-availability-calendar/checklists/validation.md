# Local Validation Checklist

- [ ] Verify `public/index.html` renders successfully in the browser.
- [ ] Confirm the session user selector loads and defaults to a valid user.
- [ ] Click a date cell and verify the selected user's availability marker appears.
- [ ] Click the same date again and verify the availability marker is removed.
- [ ] Select a different user and confirm availability markers display with a unique color.
- [ ] Verify multiple users on the same date show distinct markers and visual differentiation.
- [ ] Use previous/next month controls and confirm the calendar updates correctly.
- [ ] Refresh the page and verify availability selections persist when storage is available.
- [ ] Open the app in a browser with storage disabled and verify the warning message appears.
